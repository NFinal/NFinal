4.垂直分割
	例如用户表：
	1.根据业务需求，把经常使用的字段放在一张表中，把不经常使用的字段放到另一张表中。
	例如：用户表Users,
	create users
	{
		id int primary key,
		nickname varchar(20),
		account varchar(20),
		password varchar(10),
		
		sex bit,
		address varchar(50),
		phone varchar(12),
		birth datetime,
		age int,
		right int,
		height int,
		weight int,
		money int,
		grade int,
		last_log datetime
	}	
	把以上表分为两张，分别为users,users_info
	create users
	{
        id int primary key,
		nickname varchar(20),
		account varchar(20),
		password varchar(10)
	}
	create users_info
	{
		id int primary key,
        users_id int,---关联字段
		sex bit,
		address varchar(50),
		phone varchar(12),
		birth datetime,
		age int,
		right int,
		height int,
		weight int,
		money int,
		grade int,
		last_log datetime
	}
	如果某张表遇到了数据库瓶颈，可以根据业务把表垂直分隔为几张表。
	这种模式最多的应用于登陆，搜索，等业务中。
	比如汽车，房产，商品其属性往往非常多，一件东西对应的数据量非常大，这时候，就要考虑表的垂直分割了。
	另外，分割出来的表，可以放在同一数据库上，也可以放在其它的数据库上，这样也可以加快数据库的读写速度。
